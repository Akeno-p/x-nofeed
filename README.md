# x-nofeed

X (Twitter) のタイムラインを表示せず、自分の発信とやりとりだけに制限した個人用Xクライアントアプリ。

## 開発背景

Xは人との繋がりや仕事の幅を広げる優れたツールですが、タイムラインにはあまり目に入れたくないコンテンツが流れてくることがあります。

「興味がない」を選択しても、しばらくするとまた同種のポストが出現します。そのたびに対処するのは純粋に疲れます。

またSNS自体の問題として、タイムラインを見始めると止まらなくなり、気づけば20分、ひどい時は1時間経っていた、ということもありました。

そこで、見たくないものを除外するのではなく、必要な機能だけに絞ったクライアントを作ればいいのでは、と思い立ったのがきっかけです。

完全に新しいSNSではなくXを選んだ理由は、ユーザーがいない場所で発信してもSNSのメリットを享受しづらいと感じたためです。

## コンセプト

このアプリが担うのは「普段使い」の部分だけを想定しています。

ポストや、リプライの返信、DMの確認などをしたいだけなのに、タイムラインで意図しない情報が目に入ってしまう。それによって疲弊してしまうことを防ぐことを目的としています。

自分から発信者にコンタクトを取る、技術トレンドについて調べる、好きな発信者のポストの確認などは、日常的に必要な行為ではないと思うので、そういった目的がある時は心してX本家を開くことを想定しています。

つまり

- **普段使い → x-nofeed**：投稿・リプライ返信・DM。見たくない情報に触れずに済む
- **情報収集・交流 → X本家**：自分からコンタクトを取る、調べ物をする。　必要な時に心して開く

※ あと、xのapi利用料金は従量課金制のため、他人の投稿の取得まで含めると月額1万円は余裕で超えてくるため金銭的にも厳しいです...

## 機能

- 自分のポストの投稿
- 自分の投稿へのリプライの確認・返信
- DMの確認・返信
- API利用料金の表示

※ いいね、リポスト、フォロー、ポストの確認もできなくなっています。

## 技術スタック

- フロントエンド：Tailwind v4.3 / jQuery 4
- バックエンド：Django 5.2　/ Python 3.13
- インフラ：ConoHa VPS
  - OS：AlmaLinux
  - webサーバー：Nginx
  - SSL証明書：Let's Encrypt
  - アプリケーションサーバー：Gunicorn
  - DB：PostgreSQL

### 選定理由

私が所属する会社のシステムで使用されている技術スタックを選定しています。
まずは実務で使っている技術をしっかり扱えるようになることを優先し、その上でモダンな技術に手を広げるべきだと考えているためです。

## X API

このアプリは [X API](https://docs.x.com) を使用して、実際のXプラットフォームと連携。クローンアプリではなく、本物のXアカウントで投稿・やりとりができる。
※ 少額ですが料金はかかります。

## メモ

- このアプリはX APIの利用規約に基づき、個人利用を目的としている
- X APIを通じて取得したデータの再販は禁止されている
